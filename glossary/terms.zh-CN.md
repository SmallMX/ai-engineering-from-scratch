# AI 工程术语表

这个术语表服务于中文翻译版课程。目标不是创造最学术的译名，而是在整套课程中保持一致、清楚、可教学。

## A

### Activation Function
- **中文译名：** 激活函数
- **常见说法：** “层与层之间的非线性函数”
- **实际含义：** 在线性层之后应用的函数，用来引入非线性。没有激活函数，再多线性层也会退化成一个线性变换。

### Adam (Optimizer)
- **中文译名：** Adam (优化器)
- **常见说法：** ““最常用的深度学习优化算法””
- **实际含义：** 一种自适应学习率的梯度下降优化算法，结合了动量（Momentum）和自适应梯度（RMSProp）的优点，对每个参数动态调整学习率。

### AdamW
- **中文译名：** AdamW
- **常见说法：** ““修复了权重衰减的 Adam””
- **实际含义：** Adam 优化器的变体，将权重衰减（Weight Decay）直接应用到权重更新公式中，而不是像传统 Adam 那样与梯度混合。这样能产生更好的泛化效果。

### Agent
- **中文译名：** 智能体
- **常见说法：** “能自己思考和行动的 AI”
- **实际含义：** 一个循环：LLM 决定下一步调用什么工具，执行工具，观察结果，然后继续。
- **翻译说明：** 保留 “agent” 的技术语境；首次出现可写作“智能体 (agent)”。

### Alignment
- **中文译名：** 对齐
- **常见说法：** “让 AI 更安全”
- **实际含义：** 让 AI 系统的行为符合人的意图、价值和偏好，包括设计者没有预想到的边界情况。

### Attention
- **中文译名：** 注意力
- **常见说法：** “AI 关注重要部分的方式”
- **实际含义：** 每个 token 根据 query 和 key 的点积相关性，对其他 token 的 value 做加权求和。

### Autograd
- **中文译名：** 自动求导
- **常见说法：** “自动算梯度”
- **实际含义：** 记录张量运算并通过反向模式微分自动计算梯度的系统。

### Autoregressive
- **中文译名：** 自回归
- **常见说法：** ““一次生成一个词的 AI””
- **实际含义：** 在预测下一个 token 时，将之前生成的所有 token 作为输入，并把当前预测的输出作为输入传入下一步。GPT、LLaMA 和 Claude 都是自回归模型。

## B

### Backpropagation
- **中文译名：** 反向传播
- **常见说法：** “神经网络学习的方法”
- **实际含义：** 用链式法则从输出层向输入层反推每个权重对误差的贡献，再按比例更新权重。

### Batch Size
- **中文译名：** 批大小
- **常见说法：** “一次处理多少样本”
- **实际含义：** 每次前向和反向传播后更新权重之前处理的训练样本数量。

## C

### Chain of Thought (CoT)
- **中文译名：** 思维链
- **常见说法：** ““让 AI 一步一步想””
- **实际含义：** 一种提示工程技术，通过在 Prompt 中要求模型展示推理过程，来提高其在数学、符号推理和多步规划任务上的表现。

### Chunking
- **中文译名：** 分块 / 分段
- **常见说法：** ““把大文档切成小碎片””
- **实际含义：** 在 RAG（检索增强生成）系统中，将整篇长文章按固定字数、段落或语义切分成更小的文本块，以便更好地进行向量编码与检索。

### CNN (Convolutional Neural Network)
- **中文译名：** 卷积神经网络
- **常见说法：** ““做图像处理的 AI””
- **实际含义：** 一种特殊的神经网络结构，利用卷积层来提取局部特征（如图像边缘或空间模式），具有平移不变性，广泛应用于计算机视觉领域。

### Context Window
- **中文译名：** 上下文窗口
- **常见说法：** “AI 能记住多少”
- **实际含义：** 单次 API 调用中可容纳的最大 token 数，包括输入和输出。它不是长期记忆。

### Contrastive Learning
- **中文译名：** 对比学习
- **常见说法：** ““拉近相同事物，推开不同事物””
- **实际含义：** 一种无监督或自监督学习方法，通过最大化相似样本对（正例）之间的相似度、最小化不相似样本对（负例）之间的相似度，来学习表示特征。CLIP 就是其典型代表。

### Cosine Similarity
- **中文译名：** 余弦相似度
- **常见说法：** ““向量比对的方法””
- **实际含义：** 通过计算两个向量夹角的余弦值来衡量它们的语义相似度，该值范围在 -1 到 1 之间。它不受向量绝对大小（长度）的影响。

### Cross-Entropy
- **中文译名：** 交叉熵
- **常见说法：** “分类损失”
- **实际含义：** 衡量两个概率分布差异的指标。语言模型里通常是正确下一个 token 的负对数概率。

### CUDA
- **中文译名：** CUDA
- **常见说法：** ““让英伟达显卡算得快的东西””
- **实际含义：** NVIDIA 推出的一种通用并行计算平台和编程模型，允许开发者使用 C/C++ 等语言直接调用 GPU 进行高性能计算。

## D

### Data Augmentation
- **中文译名：** 数据增强
- **常见说法：** ““通过翻转旋转造出更多图片数据””
- **实际含义：** 通过对已有训练样本进行随机裁剪、旋转、翻转、加噪或同义词替换等操作，人为扩大数据集规模，提升模型的泛化能力。

### Decoder
- **中文译名：** 解码器
- **常见说法：** “输出部分”
- **实际含义：** Transformer 中使用因果自注意力的模块，每个位置只能关注自己之前的位置。

### Diffusion Model
- **中文译名：** 扩散模型
- **常见说法：** “从噪声生成图像的 AI”
- **实际含义：** 学习反转逐步加噪过程的生成模型。

### DPO (Direct Preference Optimization)
- **中文译名：** 直接偏好优化
- **常见说法：** ““省去奖励模型的对齐方法””
- **实际含义：** 一种人类偏好对齐方法，完全跳过了强化学习（RLHF）中训练奖励模型的步骤，直接使用包含好/坏响应的偏好对数据来优化语言模型。

### Dropout
- **中文译名：** 暂退法 / 随机失活
- **常见说法：** ““随机关闭神经元””
- **实际含义：** 在训练过程中随机把一定比例的神经元激活值设为零，防止模型过度依赖某些特定特征，起到正则化和防止过拟合的作用。在推理阶段该机制会关闭。

## E

### Eigenvalue
- **中文译名：** 特征值
- **常见说法：** ““主成分分析中的数学概念””
- **实际含义：** 在线性代数中，对于矩阵 A，若满足 Av = \lambda v（v 为非零向量），则 \lambda 为特征值。它代表了矩阵在对应特征向量方向上的缩放程度，主成分分析（PCA）中大特征值对应高方差方向。

### Embedding
- **中文译名：** 嵌入
- **常见说法：** “把词变成数字”
- **实际含义：** 把离散对象映射到连续向量空间的学习表示，相似对象在空间中更接近。

### Encoder
- **中文译名：** 编码器
- **常见说法：** “输入部分”
- **实际含义：** Transformer 中使用双向自注意力的模块，适合理解类任务。

### Epoch
- **中文译名：** 轮次
- **常见说法：** “完整看一遍数据”
- **实际含义：** 训练集中所有样本被完整处理一次。

## F

### Feature
- **中文译名：** 特征
- **常见说法：** ““数据表中的一列””
- **实际含义：** 描述数据的某个独立可测量的属性。在传统机器学习中，特征需要人工提取；而在深度学习中，模型能够从原始数据中自动学习和组合多层特征。

### Few-Shot
- **中文译名：** 少样本提示
- **常见说法：** ““在提示词里多给几个例子””
- **实际含义：** 在向语言模型提出问题前，在 Prompt 中提供少量（通常为 3-5 个）输入-输出的示例，以便模型通过上下文模式匹配理解期望的格式与任务。

### Fine-tuning
- **中文译名：** 微调
- **常见说法：** ““用自己的数据重新训练 AI””
- **实际含义：** 在预训练模型的基础上，使用较小规模的特定任务数据集继续进行训练以微调权重。它微调现有权重以适应新任务，但不适用于从头写入大量新领域知识。

### Function Calling
- **中文译名：** 函数调用
- **常见说法：** ““让 AI 决定调用哪个工具””
- **实际含义：** 一种使 LLM 能够与外部工具进行交互的结构化方法。通过向模型提供工具的 JSON Schema 定义，模型在需要时输出包含目标函数名及对应参数的 JSON 对象，由代码执行该函数并把结果回传给模型。

## G

### GAN (Generative Adversarial Network)
- **中文译名：** 生成对抗网络
- **常见说法：** ““两个 AI 左右互搏””
- **实际含义：** 一种由生成器和判别器组成的生成模型。生成器试图生成逼真数据，判别器试图区分真实数据与生成数据，两者在对抗训练中共同提升。

### GPT
- **中文译名：** GPT
- **常见说法：** ““ChatGPT 原理””
- **实际含义：** Generative Pre-trained Transformer（生成式预训练 Transformer）的缩写。一种基于仅解码器（Decoder-only）架构的语言模型，在大规模无监督文本语料库上训练以预测下一个 token。

### Gradient
- **中文译名：** 梯度
- **常见说法：** “斜率”
- **实际含义：** 由偏导数组成的向量，指向函数增长最快的方向。

### Gradient Descent
- **中文译名：** 梯度下降
- **常见说法：** “AI 改进自己的方法”
- **实际含义：** 沿着梯度相反方向调整参数，让损失函数下降的优化算法。

### Guardrails
- **中文译名：** 防护栏 / 护栏
- **常见说法：** ““安全过滤器””
- **实际含义：** 包裹在 LLM 输入和输出外层的安全校验与过滤层，用来拦截有害内容、提示词注入、敏感隐私泄露或离题回答。

## H

### Hallucination
- **中文译名：** 幻觉
- **常见说法：** ““AI 在一本正经地胡说八道””
- **实际含义：** 语言模型生成了听起来合理但实际上并不符合客观事实、训练数据或上下文输入的信息。这是因为模型是在做概率模式补全，而非进行事实检索。

### Hyperparameter
- **中文译名：** 超参数
- **常见说法：** “训练前要调的设置”
- **实际含义：** 训练前设定、控制训练过程的值，例如学习率、批大小、层数和 dropout 率。

## I

### Inductive Bias
- **中文译名：** 归纳偏置
- **常见说法：** ““模型架构中的先验假设””
- **实际含义：** 模型为了推广到未见过的数据而做出的架构假设。例如，CNN 假设像素局部相关（平移不变性），RNN 假设顺序重要，Transformer 假设一切位置均可能存在关联。

### Inference
- **中文译名：** 推理
- **常见说法：** ““运行 AI / 运行模型””
- **实际含义：** 在生产环境中使用训练好的模型输入新数据并获取预测结果的过程，在此过程中模型权重不进行更新。

## J

### JAX
- **中文译名：** JAX
- **常见说法：** ““谷歌出的新深度学习框架””
- **实际含义：** 支持高性能数值计算的 Python 库，集成了 autodiff（自动求导）、XLA（JIT 编译）和多设备并行计算。采用纯函数式编程风格。

## K

### KV Cache
- **中文译名：** 键值缓存 (KV 缓存)
- **常见说法：** ““让大模型生成变快的缓存””
- **实际含义：** 在自回归文本生成过程中，将先前 token 的 Key 和 Value 矩阵缓存起来，避免在每次生成新 token 时重复计算已输入序列，极大降低计算量。

## L

### Latent Space
- **中文译名：** 隐空间 / 潜空间
- **常见说法：** ““AI 内部的抽象表示空间””
- **实际含义：** 一种经模型学习后降低维度的特征空间，输入数据的核心流形结构和语义关系被编码在该空间中的邻近位置。

### Learning Rate
- **中文译名：** 学习率
- **常见说法：** ““AI 更新参数的速度””
- **实际含义：** 梯度下降算法中的一个超参数，用来控制在每次迭代中参数沿着负梯度方向更新的步长。

### LLM (Large Language Model)
- **中文译名：** 大语言模型
- **常见说法：** ““大模型””
- **实际含义：** 拥有数亿到上千亿参数，使用 Transformer 架构在海量文本上进行预训练的语言模型，能够理解和生成自然语言。

### LoRA (Low-Rank Adaptation)
- **中文译名：** 低秩适应 (LoRA)
- **常见说法：** ““低成本微调大模型的方法””
- **实际含义：** 一种高效参数微调技术，冻结原模型的所有权重，只在旁路中添加可训练的低秩矩阵（Rank A 和 B），极大地减少了微调所需的显存和保存的参数量。

### Loss Function
- **中文译名：** 损失函数
- **常见说法：** “模型错得有多离谱”
- **实际含义：** 把预测与目标之间的差异转换成一个可优化数值的函数。

## M

### MCP (Model Context Protocol)
- **中文译名：** 模型上下文协议 (MCP)
- **常见说法：** ““大模型和工具连接的标准接口””
- **实际含义：** 由 Anthropic 主导的一项开放标准，旨在提供一种通用的客户端-服务器协议，以便安全、标准地让大语言模型访问外部数据源、本地文件和工具。

### Mixed Precision
- **中文译名：** 混合精度训练
- **常见说法：** ““半精度和单精度混合加速””
- **实际含义：** 在训练过程中同时使用 16 位浮点数（FP16/BF16）和 32 位浮点数（FP32）。前向和反向传播使用半精度以加速计算并节省显存，权重更新和主权重使用单精度以保证精度稳定。

### MoE (Mixture of Experts)
- **中文译名：** 混合专家架构 (MoE)
- **常见说法：** ““多个小模型拼成的大模型””
- **实际含义：** 一种稀疏神经网络结构，其中包含多个独立的“专家”网络，并有一个门控网络决定将每个输入路由给哪个或哪几个专家处理。这样可以在大幅增加参数量的同时，保持较低的单次计算成本。

## N

### NaN (Not a Number)
- **中文译名：** 非数 (NaN)
- **常见说法：** ““梯度爆炸算崩了的值””
- **实际含义：** 表示未定义或无法表示的数值结果。在深度学习中，通常是因为学习率过高、梯度爆炸或除以零，导致权重和损失变成了 NaN，使训练失效。

### Normalization
- **中文译名：** 归一化 / 标准化
- **常见说法：** ““把数据变成统一的量纲””
- **实际含义：** 通过缩放和移动数值，使数据分布具有零均值、单位方差，或将取值范围限制在 [0, 1] 之间。这有助于加速模型收敛并避免数值溢出，如 Batch Norm 和 Layer Norm。

## O

### Optimizer
- **中文译名：** 优化器
- **常见说法：** ““更新权重的算法””
- **实际含义：** 根据损失函数计算出的梯度，决定如何调整模型权重参数的数学算法。常见的优化器包括 SGD、Adam、AdamW。

### Overfitting
- **中文译名：** 过拟合
- **常见说法：** ““死记硬背””
- **实际含义：** 模型在训练集上表现极好（记住了噪声和细节），但在未见过的测试集或新数据上表现很差，缺乏泛化能力。

## P

### Parameter
- **中文译名：** 参数
- **常见说法：** ““模型的权重和偏置””
- **实际含义：** 在训练过程中由优化算法从数据中学习到的模型内部变量，主要是连接权重（Weights）和偏置（Biases）。

### Perplexity
- **中文译名：** 困惑度 (PPL)
- **常见说法：** ““大模型预测词的纠结程度””
- **实际含义：** 衡量语言模型质量的指标。它等于测试集负对数似然的指数，直观上表示模型在预测下一个 token 时需要从多少个备选词中做选择。困惑度越低，说明模型预测越准确。

### Precision & Recall
- **中文译名：** 精确率与召回率
- **常见说法：** ““查准率与查全率””
- **实际含义：** 分类任务的两个评估指标。精确率指预测为正例的样本中真正为正例的比例；召回率指所有真实正例中被预测出来的比例。两者通常需要权衡。

### Prompt Engineering
- **中文译名：** 提示词工程
- **常见说法：** ““写好提示词的技巧””
- **实际含义：** 设计、编写和优化输入给大语言模型的提示文本，以引导模型输出更符合预期、更准确回答的实践学问。

### Prompt Injection
- **中文译名：** 提示词注入
- **常见说法：** ““绕过 AI 限制的越狱指令””
- **实际含义：** 一种针对 LLM 系统的攻击方式，用户通过构造特殊的输入文本（绕过系统设定），诱导或欺骗模型忽略系统原生 Prompt，从而执行非授权的操作或输出受限内容。

## Q

### QLoRA
- **中文译名：** QLoRA
- **常见说法：** ““量化版的 LoRA””
- **实际含义：** 一种极端高效的微调方法，首先将预训练的 LLM 量化为 4-bit 浮点精度（NF4 格式），然后在其上通过 LoRA 进行参数微调。它在保持预测能力的同时大幅降低显存需求。

### Quantization
- **中文译名：** 量化
- **常见说法：** ““降低模型权重的精度””
- **实际含义：** 将模型参数从高精度格式（如 FP32 或 FP16）转换为低精度格式（如 INT8 或 INT4）的过程。这样可以极大减少显存占用并加速推理，同时尽可能减小精度损失。

## R

### RAG (Retrieval-Augmented Generation)
- **中文译名：** 检索增强生成
- **常见说法：** ““给大模型配个知识库””
- **实际含义：** 在将问题发送给大模型前，先从外部数据源（如向量数据库）检索出相关文档，并将这些上下文拼接到 Prompt 中，以增强大模型生成回答的事实准确性和时效性。

### ReLU
- **中文译名：** ReLU 激活函数
- **常见说法：** ““最简单的非线性层””
- **实际含义：** 线性整流函数（Rectified Linear Unit），公式为 `f(x) = max(0, x)`。当输入大于零时直接输出，小于零时输出为零。计算极其简单，并且在正区间内不存在梯度消失问题。

### RLHF (Reinforcement Learning from Human Feedback)
- **中文译名：** 人类反馈强化学习
- **常见说法：** ““用人类的打分调教大模型””
- **实际含义：** 一种对齐训练技术。首先收集人类对模型输出的偏好并训练一个奖励模型，然后使用 PPO 等强化学习算法调整大语言模型的参数，使其输出更符合人类期望。

### ROUGE
- **中文译名：** ROUGE (评估指标)
- **常见说法：** ““评估摘要写得好不好的指标””
- **实际含义：** 主要用于评估文本摘要和翻译质量的指标。通过计算模型生成的文本与人类参考答案之间重合的 n-gram（n元组）数量来评分。

## S

### Self-Attention
- **中文译名：** 自注意力机制
- **常见说法：** ““词与词互相计算关联度””
- **实际含义：** Transformer 的核心机制。通过对输入序列中的每个 token 分别计算 Query、Key 和 Value 向量，并用 Q 和 K 的点积注意力分数对 Value 进行加权求和，使每个 token 能够捕捉到序列中所有其他位置的语义关联。

### Semantic Search
- **中文译名：** 语义搜索
- **常见说法：** ““理解意思的搜索””
- **实际含义：** 不同于关键字匹配，语义搜索通过将查询词和文档映射到相同的向量空间中，计算向量之间的几何距离，从而找出表达相同或相近概念的文档。

### SFT (Supervised Fine-Tuning)
- **中文译名：** 监督微调 (SFT)
- **常见说法：** ““给模型喂指令和回答的训练””
- **实际含义：** 在基座模型预训练之后，使用由“指令-回答”组成的配对数据进行有监督微调，从而使模型学会像助手一样遵循用户指令和进行对话。

### Softmax
- **中文译名：** Softmax 函数
- **常见说法：** ““把输出转换成概率值””
- **实际含义：** 将一个实数向量映射为一个概率分布。它对向量中的每个元素进行指数化，然后进行归一化，使得所有输出值在 [0, 1] 之间且总和为 1。

### Streaming
- **中文译名：** 流式传输 / 打字机效果
- **常见说法：** ““AI 边算边输出””
- **实际含义：** 在生成文本时，模型每预测出一个 token 就立即通过网络发送给客户端，使其无需等待整段长文本生成完毕即可开始逐步显示，缩短了首字延迟（TTFT）。

### Swarm
- **中文译名：** 群体智能 / 智能体群
- **常见说法：** ““多个智能体协同工作””
- **实际含义：** 多个独立智能体协同工作的系统形态，其中任务通过智能体之间的路由、分工与协作，共同拼凑出复杂的端到端解决方案。

### System Prompt
- **中文译名：** 系统提示词
- **常见说法：** ““AI 扮演什么角色的设定””
- **实际含义：** 在对话开始前提供给模型的最高优先级指令，用来定义模型的角色定位、回复风格、行为约束以及不可逾越的安全红线。

## T

### Temperature
- **中文译名：** 温度
- **常见说法：** ““控制 AI 回答的创造力和随机性””
- **实际含义：** 用于调整 Softmax 概率分布平滑度的超参数。温度越高，概率分布越平缓，输出的随机性和多样性（创造力）增加；温度越低，分布越陡峭，输出越倾向于选择概率最高的词，确定性增强。

### Tensor
- **中文译名：** 张量
- **常见说法：** ““多维数组””
- **实际含义：** 深度学习中的基本数据结构，表现为包含数值的 n 维数组。标量是 0 维张量，向量是 1 维张量，矩阵是 2 维张量，多维数组则是更高维的张量。

### Token
- **中文译名：** Token / 标记 / 词元
- **常见说法：** ““AI 识别的小字块””
- **实际含义：** 文本经分词器（Tokenizer）处理后分割成的最小单位。一个 Token 可以是一个完整的单词、单词的一部分（如 subword 词片）或单个字符。大模型的计费和上下文限制都是基于 Token 数量的。

### Transfer Learning
- **中文译名：** 迁移学习
- **常见说法：** ““借用已有知识””
- **实际含义：** 将在一个任务上学习到的特征与知识应用到另一个相关但不同的新任务上的机器学习范式。微调大语言模型就是一种典型的迁移学习。

### Transformer
- **中文译名：** Transformer 架构
- **常见说法：** ““现代大模型的底层引擎””
- **实际含义：** 2017 年提出的基于自注意力机制的深度学习架构，彻底摒弃了循环和卷积结构，能够实现高效的并行训练并捕获长距离上下文依赖，是 GPT、Gemini 和 Claude 等几乎所有主流 LLM 的底层基石。

## U

### Underfitting
- **中文译名：** 欠拟合
- **常见说法：** ““模型没学明白””
- **实际含义：** 模型太简单，以至于在训练集和测试集上都表现不佳，没有学到数据背后的真实规律。

## V

### VAE (Variational Autoencoder)
- **中文译名：** 变分自编码器
- **常见说法：** ““用于生成图像的概率自编码器””
- **实际含义：** 一种生成模型，通过编码器将输入映射为隐空间中的概率分布均值和方差，再从该分布中采样并通过解码器重建输入，可用于生成全新的数据样本。

### Vector Database
- **中文译名：** 向量数据库
- **常见说法：** ““存放 AI 记忆的数据库””
- **实际含义：** 专为高维向量（如 Embedding）存储和快速相似性检索（如 ANN 近邻搜索）而设计的数据库。是 RAG 系统必不可少的底层基础设施。

## W

### Weight
- **中文译名：** 权重
- **常见说法：** “某个输入有多重要”
- **实际含义：** 缩放输入贡献的乘数，训练过程会不断调整它。

### Weight Decay
- **中文译名：** 权重衰减
- **常见说法：** ““限制权重过大防止过拟合的正则化””
- **实际含义：** 一种正则化技术，通过在损失函数中增加参数的 L2 范数惩罚项，促使优化算法将权重大小拉向零，从而限制模型复杂度，防止过拟合。

## Z

### Zero-Shot
- **中文译名：** 零样本提示
- **常见说法：** ““直接问 AI，不给例子””
- **实际含义：** 在 Prompt 中直接给语言模型描述要完成的任务并提问，而不提供任何输入-输出的示例，完全依赖模型原生的泛化和推理能力。
