# AI 工程术语表

这个术语表服务于中文翻译版课程。目标不是创造最学术的译名，而是在整套课程中保持一致、清楚、可教学。

## A

### Agent
- **中文译名：** 智能体
- **常见说法：** “能自己思考和行动的 AI”
- **实际含义：** 一个循环：LLM 决定下一步调用什么工具，执行工具，观察结果，然后继续。
- **翻译说明：** 保留 “agent” 的技术语境；首次出现可写作“智能体 (agent)”。

### Attention
- **中文译名：** 注意力
- **常见说法：** “AI 关注重要部分的方式”
- **实际含义：** 每个 token 根据 query 和 key 的点积相关性，对其他 token 的 value 做加权求和。

### Alignment
- **中文译名：** 对齐
- **常见说法：** “让 AI 更安全”
- **实际含义：** 让 AI 系统的行为符合人的意图、价值和偏好，包括设计者没有预想到的边界情况。

### Activation Function
- **中文译名：** 激活函数
- **常见说法：** “层与层之间的非线性函数”
- **实际含义：** 在线性层之后应用的函数，用来引入非线性。没有激活函数，再多线性层也会退化成一个线性变换。

### Autograd
- **中文译名：** 自动求导
- **常见说法：** “自动算梯度”
- **实际含义：** 记录张量运算并通过反向模式微分自动计算梯度的系统。

## B

### Backpropagation
- **中文译名：** 反向传播
- **常见说法：** “神经网络学习的方法”
- **实际含义：** 用链式法则从输出层向输入层反推每个权重对误差的贡献，再按比例更新权重。

### Batch Size
- **中文译名：** 批大小
- **常见说法：** “一次处理多少样本”
- **实际含义：** 每次前向和反向传播后更新权重之前处理的训练样本数量。

### Bias
- **中文译名：** 偏置
- **常见说法：** “阈值”
- **实际含义：** 加到加权和上的常数项，用来平移决策边界。

## C

### Chain of Thought
- **中文译名：** 思维链
- **常见说法：** “让模型一步一步想”
- **实际含义：** 一种提示技术，通过要求模型展示推理步骤来提升多步问题表现。

### Context Window
- **中文译名：** 上下文窗口
- **常见说法：** “AI 能记住多少”
- **实际含义：** 单次 API 调用中可容纳的最大 token 数，包括输入和输出。它不是长期记忆。

### Cross-Entropy
- **中文译名：** 交叉熵
- **常见说法：** “分类损失”
- **实际含义：** 衡量两个概率分布差异的指标。语言模型里通常是正确下一个 token 的负对数概率。

## D

### Decision Boundary
- **中文译名：** 决策边界
- **常见说法：** “分类器切换判断的地方”
- **实际含义：** 把输入空间分成不同类别区域的边界；在线性模型中通常是 `w*x + b = 0`。

### Decoder
- **中文译名：** 解码器
- **常见说法：** “输出部分”
- **实际含义：** Transformer 中使用因果自注意力的模块，每个位置只能关注自己之前的位置。

### Diffusion Model
- **中文译名：** 扩散模型
- **常见说法：** “从噪声生成图像的 AI”
- **实际含义：** 学习反转逐步加噪过程的生成模型。

## E

### Embedding
- **中文译名：** 嵌入
- **常见说法：** “把词变成数字”
- **实际含义：** 把离散对象映射到连续向量空间的学习表示，相似对象在空间中更接近。

### Encoder
- **中文译名：** 编码器
- **常见说法：** “输入部分”
- **实际含义：** Transformer 中使用双向自注意力的模块，适合理解类任务。

### Epoch
- **中文译名：** 轮次
- **常见说法：** “完整看一遍数据”
- **实际含义：** 训练集中所有样本被完整处理一次。

## G

### Gradient
- **中文译名：** 梯度
- **常见说法：** “斜率”
- **实际含义：** 由偏导数组成的向量，指向函数增长最快的方向。

### Gradient Descent
- **中文译名：** 梯度下降
- **常见说法：** “AI 改进自己的方法”
- **实际含义：** 沿着梯度相反方向调整参数，让损失函数下降的优化算法。

## H

### Hyperparameter
- **中文译名：** 超参数
- **常见说法：** “训练前要调的设置”
- **实际含义：** 训练前设定、控制训练过程的值，例如学习率、批大小、层数和 dropout 率。

## L

### Linearly Separable
- **中文译名：** 线性可分
- **常见说法：** “能画一条线分开”
- **实际含义：** 存在一个线性边界，可以把不同类别完全分开的数据性质。

### Loss Function
- **中文译名：** 损失函数
- **常见说法：** “模型错得有多离谱”
- **实际含义：** 把预测与目标之间的差异转换成一个可优化数值的函数。

## M

### Multi-layer Perceptron
- **中文译名：** 多层感知机
- **常见说法：** “真正的神经网络”
- **实际含义：** 把多个感知机或神经元按层堆叠，前一层输出作为后一层输入的网络。

## P

### Perceptron
- **中文译名：** 感知机
- **常见说法：** “假的神经元”
- **实际含义：** 一个线性分类器：输入与权重点积，加上偏置，再通过阶跃函数输出二分类结果。

## T

### Tokenization
- **中文译名：** 分词
- **常见说法：** “把文本切成小块”
- **实际含义：** 把文本切分成模型处理的 token 序列，token 可以是词、子词、字符或字节片段。

## W

### Weight
- **中文译名：** 权重
- **常见说法：** “某个输入有多重要”
- **实际含义：** 缩放输入贡献的乘数，训练过程会不断调整它。
